Primary Git Main Porcelain Commands

This page lists some of the main git high level commands and what they do

  1. Git Add
    • Adds a file to the index or staging area.
    • For example,

      git add index.html

      or

      git add .

  2. Git Clone
    • Copies a repository from an existing URL.
    • For example,

      git clone https://github.com/example/example.git

  3. Git Commit
    • Records a snapshot of the file permanently in the version history.
    • For example,

      git commit -m 'comment explaining changes since last commit '

  4. Git Config
    • Sets the author name and email address to be used with the commit command.
    • For example,

      git config --global user.name 'username '

    • For example,

      git config --global user.email 'user@email.com '

  5. Git Fetch
    • Checks for available changes from the remote repository, but does not actually transfer files.
    • This is useful to know what changes you will see before performing a pull.
    • For example,

      git fetch

  6. Git Merge
    • Merges the specified branch's history into the current branch.
    • For example,

      git merge branchName

  7. Git Move
    • Moves a file from its current path to a new path.
    • For example,

      git mv currentPath newPath

  8. Git Pull
    • Checks for and brings available changes from the remote repository into your local copy.
    • For example,

      git pull https://github.com/example/example.git

  9. Git Push
    • Sends the committed changes of the master branch (or other specified branch) to the associated remote repository.
    • For example,

      git push variableName branchName

  10. Git Remote
    • Connects your local repository to the remote server.
    • For example,

      git remote add variableName https://github.com/example/example.git

  11. Git Remove
    • Removes individual files or a collection of files from the staging area and the working directory of the current branch.
    • For example,

      git rm exampleFile.txt

  12. Git Status
    • Lists all files, stating files that have changes to be committed, or files that need to be added to staging area.
    • For example,

      git status
